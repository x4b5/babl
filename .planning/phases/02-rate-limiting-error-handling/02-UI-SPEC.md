## Copywriting Contract

| Element                       | Copy                                           |
| ----------------------------- | ---------------------------------------------- |
| Rate limit (countdown active) | "Overbelast. Wacht {N}s..."                    |
| Rate limit (no countdown)     | "Overbelast. Even geduld."                     |
| Timeout error                 | "Duurt te lang — probeer een korter fragment." |
| Network error                 | "Geen internet."                               |
| Upstream disconnect           | "Backend niet bereikbaar."                     |
| Generic fallback              | "Fout bij verwerking."                         |

**Tone:** Short and direct. No apologies ("Sorry", "Helaas"). No technical jargon (no HTTP status codes, no "API", no "server"). Actionable where possible.

**Source:**

- D-09 from CONTEXT.md: "Kort en direct — geen 'Helaas', 'Sorry', of overbodige woorden"
- D-10 from CONTEXT.md: "Gebruiksvriendelijk Nederlands, geen technische termen"
- RESEARCH.md error taxonomy messages (lines 579-585)

**Countdown format:**

- Template: `"Overbelast. Wacht {countdown}s..."`
- Updates every second (live counter)
- Example sequence: "Overbelast. Wacht 30s..." → "Overbelast. Wacht 29s..." → ... → "Overbelast. Wacht 1s..." → (auto-retry, message clears)

**Auto-retry behavior (D-03):**

- Countdown reaches 0 → error message clears → API call automatically retries
- No user action required
- If retry fails with 429 again, countdown restarts (max 3 cycles recommended per RESEARCH.md Open Question 1)

## Interaction Patterns

### Error Display

**Location:** Inline in existing `error` element (line 1388-1395 of `+page.svelte`). No new UI components.

**States:**

1. **Hidden** (`error === ''`): No error element rendered
2. **Retry-able** (`error_type === 'rate_limit'`): Amber border/background, countdown text updates every 1s
3. **Fatal** (other error types): Red border/background, static text

**Animation:** Existing `animate-slide-up` (from `app.css` line 568-577). No new animations needed.

**Countdown Timer:**

- JavaScript `setInterval` with 1-second interval
- Svelte `$state` variable `countdown` tracks remaining seconds
- Cleanup via `$effect` return function (prevents memory leaks)
- Pattern from RESEARCH.md lines 525-625

**Source:** D-01 from CONTEXT.md: "Inline error display — countdown verschijnt in het bestaande `error` element, geen nieuwe UI-componenten."

## Accessibility

### Reduced Motion

All animations respect `prefers-reduced-motion` via existing media query in `app.css` lines 17-24. Countdown timer text updates are not animations — no reduced motion override needed.

### Semantic HTML

Error alert uses `<div>` with ARIA implicit role. Consider adding `role="alert"` for screen reader announcement when error appears.

**Recommendation:** Add `role="alert"` and `aria-live="assertive"` to error div to ensure screen readers announce new errors immediately.

## Design Decisions From Context

### Locked Decisions (from 02-CONTEXT.md)

Pre-populated from user decisions. These are NOT research recommendations — they are requirements.

1. **D-01:** Inline error display in existing `error` element (no new UI components)
2. **D-02:** Live countdown updates every second
3. **D-03:** Auto-retry when countdown reaches 0
4. **D-06:** Visual distinction: amber for retry-able, red for fatal
5. **D-09:** Short direct copy ("Overbelast. Wacht 30s." not "Helaas, de API is overbelast")
6. **D-11:** Text only, no action buttons

### Inherited Patterns (from Phase 1)

7. **D-08 (Phase 1):** User-friendly Dutch, no technical terms
8. **D-09 (Phase 1):** Consistent tone across all error messages

### Claude's Discretion (from 02-CONTEXT.md)

These were open to research. Recommendations:

- **Exact amber/red color values:** Amber = `#f59e0b` (border-amber-500), Red = `#ef4444` (border-red-500). Both at 20% border opacity, 10% background opacity. (Matches existing patterns in app.css and +page.svelte)
- **Auto-retry loop limit:** Max 3 countdown cycles. After 3rd failure, show final error "Nog steeds overbelast. Probeer later handmatig." without auto-retry. (Prevents infinite loops if API down for extended period)
