<objective>
Fix offset filtering boundary bug and add SSE stream timeout detection.

Purpose: Two distinct bugs remain. (1) Audio segments spanning the offset boundary are silently dropped because the filter uses `start >= offset` instead of `end > offset - tolerance`, causing missing words in live transcription. (2) SSE streams (local transcription and correction) can stall indefinitely with no timeout, showing an eternal spinner. This plan fixes the offset filter with a 0.5s tolerance window, adds frontend timestamp deduplication to handle overlapping segments, and wraps SSE fetch calls with 30-second AbortSignal timeout.

Output: Fixed offset filter in backend, deduplication logic in frontend, SSE timeout on both sendAudioLocal and fetchCorrection functions.
</objective>
