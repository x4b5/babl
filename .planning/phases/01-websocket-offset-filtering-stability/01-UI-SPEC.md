## Phase 1 UI Changes

### New Component: Reconnection Banner

**Visual specification:**

- **Position:** Inline banner above transcription text area (not modal, not toast)
- **Styling:** Glass morphism effect using existing `.glass` class
- **Background:** `rgba(255, 255, 255, 0.08)` (stronger glass for visibility)
- **Border:** `1px solid rgba(255, 255, 255, 0.12)` with `border-l-4` accent in `#d4ff00` (neon)
- **Border radius:** `0.5rem` (8px)
- **Padding:** `12px 16px` (py-3 px-4)
- **Animation:** `animate-fade-in` on mount (0.5s ease-out)
- **Typography:** `text-sm` (14px), `font-normal` (400), color `#e2e8f0`

**States:**

1. **Reconnecting (attempts 1-4):**
   - Copy: "Verbinding herstellen (poging {N}/5)..."
   - Icon: Spinning loader (existing `.animate-spin-slow` class)
   - Border accent: `#d4ff00` (neon lime)

2. **Permanent failure (attempt 5):**
   - Copy: "Verbinding verloren. Je opname is bewaard — gebruik Bestand Upload om alsnog te transcriberen."
   - Icon: None
   - Border accent: `#ef4444` (red)
   - Banner remains visible until user dismisses or navigates away

3. **Success (reconnected):**
   - Banner auto-hides with `animate-fade-in` reverse (opacity 0)
   - Duration: 2 seconds after reconnection

**Behavior:**

- Non-dismissible during reconnection attempts (states 1-4)
- Auto-hide on success (state 3)
- Permanent failure message persists until user takes action (state 2)

**Integration point:** Insert banner conditionally in `src/routes/transcribe/+page.svelte` above the `liveSegments` / `partialText` display area, reactive to `reconnecting` and `reconnectStatus` state variables (declared in 01-RESEARCH.md WebSocket reconnection code example).

**Source:** Locked decision D-01 from 01-CONTEXT.md. Styling derived from existing `.glass` pattern and Tailwind utilities in `src/app.css`.

### No Other Visual Changes

All other Phase 1 changes are backend logic or invisible deduplication:

- WebSocket heartbeat: backend only, no UI
- Offset filtering tolerance: invisible to user
- SSE timeout detection: reuses existing error display pattern
- Timestamp deduplication: invisible to user
